Example 6 — Positioning schemes

Some of these examples are faked a little bit to illustrate things better. The positioned box in the right example of each pair also has the styles top: -1em; left: -1em;, and is slightly transparent to more easily see what’s going on.

position: static;

Text before a block with position: static;.

position: static;

This is the default positioning scheme.

Static blocks are laid out in the “normal flow”.

position: static;

Positioning with top, left etc. has no effect.

position: relative;

Relative blocks are like static blocks.

position: relative;

They create a containing block for positioned and floated children.

They take up space in the normal flow.

position: relative;
position: relative;

But they can be moved from their static position with top, left, etc.

position: absolute;

Absolute blocks are removed from the flow.

position: absolute;

They are positioned relative to their containing block.

They overlap other elements.

position: absolute;

They are positioned relative to their containing block.

position: fixed;

Fixed blocks are like absolute blocks.

position: fixed;

For print styles they appear on each page.

But they’re positioned relative to the viewport.

position: fixed;

For print styles they appear on each page.
